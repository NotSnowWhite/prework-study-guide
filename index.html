HTML

  • We can use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage
  • Head element: contains information about the webpage
  • Body element: represents the visible content shown to the user
  • Section element: designates and separates into sections for content

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • Inline: style an element directly in the HTML file
  • Internal: embed an entire stylesheet directly in the HTML file using a -style- element in the -head- element
  • External: keep all the CSS rules in a separate file. When using an external style sheet, we link the file using a -link- element placed in the HTML's -head-
  • Selector: defines the element or attributes to which the rules, or declarations, will apply
  • Declarations: contain two important components: the CSS property we want to apply and the value of the property. For example, color: blue
  • Margin property: indicates how much space we want around the outside of an element. The margin is the transparent buffer outside an element
  • Padding property: adds space around the content inside an element. The padding, while transparent, is located inside the element
  • * selector: apply a rule to all the elements visible on the page
  • Combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors. For example, header, footer. Rather than repeating a rule which would add more lines of code and repetition, two or more elements that will share the same styling can be grouped together. This helps us write Do Not Repeat Yourself (DRY) code
  • Text-align: positioning property that centers a title or heading on a webpage
  • -img- selector: assigns property values to any image element in the markup
  • display property assigned with the block value: assigns each image to start on a new line
  • -link- element: creates the connection from the CSS file to the external file
  • rel attribute: specifies the relationship between the current document and the linked document resource
  • href attribute: specifies the location (URL) of the external resource. The href attribute uses a relative path, meaning the location is derived from the current location
  • Class attributes: allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector. Then the class attribute can be assigned to the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule
  • Class selector: any HTML element assigned to the class selector will gain those property assignments from the CSS. For example, .card. The period card before designates it a class selector and any html element with the .card class will receive the assigned properties

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stages changes to add the changed files in our working directory to the staging area of Git
      • add -A: commits all files to the working branch
  • git commit -m: commits changes to the repository so that it is up to date with our working directory
      • -m: associates a message to the commit. Commit messages are short, descriptive messages of the changes implemented
  • git pull origin main: sync local and remote branches; pull the base branch into your feature branch
  • git push origin feature/name: push up all our changes
  • It's important to frequently sync the local version of your code with the remote version in GitHub

JavaScript

  • Javascript is case sensitve
  • A semicolon(;) at the end of a line indicates where a statement ends. It is only required when you need to separate statements on a single line
  • Control flow: the order in which a computer executes code in a JavaScript file
  • Script element: applies the JavaScript to the page(like the link element for CSS), so it can have an effect on the HTML (along with the CSS, and anything else on the page)
  • Variables: a named container that holds or store values. For example, myVariable = "Gabi". "Gabi" is the value
  • const keyword: allows us to create variables that can't be reassigned a value. It essentially "locks down" what that variable will be, and once we declare it, we can never reassign it
  • let keyword: allows us to reassign the value of the variable over time
  • Types of Variables Data Types:
      • String: a sequence of text enclosed in single or double quotation marks
      Number: numbers. These do not need quotation marks
      Boolean: a true/false value that does not need quotation marks
      Array: a structure that allows you to store multiple values in a single reference. For example, let myVariable = [1,'Bob','Steve',10]; Refer to each member of the array as myVariable[0], myVariable[1], etc
      Object: This can be anything. Everything in JavaScript is an object and can be stored in a variable
  • Console.log method: output a message to the web console by adding an argument
      • Console: is a test environment to check on code
      .log(): is a set of instructions for the computer to follow. Whatever is put inside the parentheses will be output
  • Comments: snippets of text that can be added along with code. Placed between /* */ or, if without link breaks, behind two slashes //
  • Operator: a mathematical symbol that produces a result based on two values (or variables)
  • Conditionals: code structures used to test if an expression returns true or not. A very common form of conditionals is the if...else statement
  • Funtions: reusable blocks of code that contain instructions for a computer to perform a specific task, but executes only when called. Means you can control where in the control flow a task is executed, or if at all. Variable name followed by parentheses is likely a function
  • Arguments: bits of data to go inside the parentheses, separated by commas if there is more than one argument
  • The code to be executed is written inside the curly brackets {}
  • Event handlers: code structures that listen for activity in the browser, and run code in response. For example, a click event which is fired when you click on something with the mouse
  • undefined indicates the absence of a value, while null indicates the absence of an object
  • conditional statements and loops: control the order in which code is executed, or if it is even executed at all; lets us alter the control flow so that code executes only if a certain condition is met
  • if statement: code will only execute if the statement meets a condition, or is true
  • Strict equality operator (===): checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal
  • Array: is a single variable that is used to hold a group of data
  • Index: a unique number that identifies the location of the data item in an array, or location number, allowing access to individual items in an array. Programs typically have an index of 0
  • A for loop: uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. we can tell the computer to go through each of the tasks in our array and log them to the console until there are no tasks left
      • Starting point: start at the beginnging of the array by declaring a variable x with a value of 0 since the first index is always 0 (let x = 0)
      Second statement - condition: as long as the number in variable x is less than the length of the array, the loop will keep running. length is a built-in array property to express the condition of executing once for each item (x < shapes.length)
      final statement: allows the array to iterate over each item. x++ is a shortcut to x+1. Now each loop, x increases by 1
  • Commenting out code: makes it inactive without deleting it, just in case you need it later. To comment out multiple lines of JavaScript code, put /* before and */ after the block of code
